# Архитектура и Структура Frontend (RescueMesh)

Этот документ предназначен для разработчиков и будущих сессий ИИ. Он содержит подробное описание структуры фронтенда, дизайн-системы, используемых компонентов и точек интеграции с будущим бэкендом.

---

## 📂 Структура проекта

Фронтенд построен на **React (TypeScript) + Vite**.

```text
demo-app/
├── package.json               # Зависимости и скрипты запуска (dev, build, preview)
├── vite.config.ts             # Конфигурация Vite с поддержкой React-плагина
├── tsconfig.json              # Настройки компилятора TypeScript
├── index.html                 # Главная точка входа HTML
├── index.tsx                  # Инициализация React (рендер компонента App)
├── main.tsx                   # Главный контейнер приложения, управление стейтом и модальными окнами
├── index.css                  # Глобальные стили, дизайн-система и переменные тем оформления
└── components/                # Директория с переиспользуемыми React-компонентами
    ├── TopBanner.tsx          # Верхняя панель статуса синхронизации и тумблер темы
    ├── Sidebar.tsx            # Боковое навигационное меню (для десктопной версии)
    ├── BottomNavbar.tsx       # Нижняя навигационная панель (для мобильной версии)
    ├── MapCard.tsx            # Карточка оффлайн-карты на главном экране
    ├── ActionButtons.tsx      # Панель быстрых экстренных кнопок (SOS и Волонтер)
    ├── AuthForm.tsx           # Матовое окно входа/регистрации (с переключением режимов)
    └── AIAdvisorCard.tsx      # Компонент-карточка ИИ (зарезервирован для будущего использования)
```

## 🎨 Световая и Темная темы (Дизайн-система)

Приложение спроектировано в строгом стиле **Apple macOS Glassmorphism** (полупрозрачные размытые панели, скругленные углы, аккуратные тени).

### Использование CSS-переменных (`index.css`):
Все цвета, фоны, границы и тени завязаны на CSS-переменные, определенные в `:root` (темная тема по умолчанию) и переопределенные в классе `body.light-theme` (светлая тема):

- `--bg-primary` — Системный фон приложения (темно-серый / светло-серый).
- `--bg-secondary` — Фон основного контента.
- `--text-primary` — Цвет основного текста (белый / черный).
- `--text-secondary` — Цвет приглушенного текста (серый).
- `--text-tertiary` — Цвет вспомогательного текста / плейсхолдеров.
- `--glass-bg`, `--glass-bg-hover` — Фон стеклянных панелей с разной степенью прозрачности.
- `--glass-border`, `--glass-border-hover` — Тонкие границы стеклянных панелей.
- `--modal-bg`, `--modal-border`, `--modal-backdrop` — Свойства модальных окон.
- `--segmented-bg`, `--segment-active-bg` — Стили переключателей (Tabs/Segments).
- `--input-bg`, `--input-border`, `--input-focus-bg` — Фоновые свойства инпутов и селекторов.
- `--grid-color`, `--grid-opacity` — Цвет и прозрачность координатной сетки оффлайн-карты.

### Переключение темы:
В компоненте `App` (`main.tsx`) объявлен стейт `isLightTheme`. При его переключении класс `light-theme` добавляется или удаляется у элемента `document.body`. Компоненты реагируют на это автоматически через CSS, а также получают проп `isLightTheme` для управления иконками (например, `TopBanner` переключает иконку Солнца/Луны).

## 🧩 Описание компонентов и логики

### 1. `main.tsx` (Центральный Хаб)
- **Управление навигацией:** Стейт `activeTab` принимает одно из пяти значений: `'MAPA'` (Карта), `'NÁVODY'` (Инструкции), `'POMOC'` (Центр помощи), `'ZPRÁVY'` (Местные каналы), `'PROFIL'` (Профиль пользователя).
- **Данные сообщений:** В константе `LATEST_MESSAGES` хранится массив чрезвычайных оповещений (тип, заголовок, описание, важность, источник, время публикации). На главном экране выводятся два последних сообщения с возможностью клика для перехода во вкладку «Сообщения».
- **Стейт пользователя:** Стейт `user` хранит информацию об авторизованном пользователе. Если пользователь не вошел, он получает объект `GUEST_USER` (режим гостя). В режиме гостя клик по кнопкам SOS или Волонтер автоматически открывает окно авторизации.
- **Модальные окна:** Стейт `modalType` управляет отображением поп-апов:
  - `SYNC` — статус оффлайн-синхронизации с соседями по mesh-сети.
  - `LEGEND` — условные обозначения на карте.
  - `MAP_FULL` — симулятор полноэкранной интерактивной карты.
  - `SOS` — подтверждение отправки экстренного сигнала.
  - `VOLUNTEER` — выбор роли и регистрация помощи.
  - `AI` — чат с оффлайн-ассистентом.
  - `AUTH` — форма авторизации.

### 2. `components/AuthForm.tsx` (Авторизация)
- Переключается между режимами `LOGIN` (Вход по номеру телефона и PIN) и `REGISTER` (Создание профиля).
- В режиме регистрации предоставляет выбор района/зоны пребывания и чекбокс волонтера с выбором специализации (физическая помощь, медик, транспорт).
- Содержит быструю кнопку-байпас для тестирования (вход под демо-профилем «Jan Dvořák»).

### 3. `components/MapCard.tsx` (Карта)
- Карточка оффлайн-карты на главном экране. Отображает стилизованную координатную сетку с прицелом и точками интереса (водозабор, медпункт).
- Клик по карточке открывает полноэкранный интерактивный симулятор карты (`MAP_FULL`).

### 4. `components/TopBanner.tsx` (Верхняя панель)
- Показывает предупреждение об оффлайн-режиме и время последней синхронизации.
- Содержит анимированную кнопку переключения темы.

### 5. `components/Sidebar.tsx` и `components/BottomNavbar.tsx` (Навигация)
- Боковая панель отображается на десктопах, нижняя — на экранах телефонов (ширина `< 768px`).

## ⚡ Интеграция с Backend (План для следующей сессии)

При интеграции фронтенда с бэкендом (или локальными демонами оффлайн-ноды) необходимо заменить заглушки на реальные запросы.

### 1. Авторизация и профиль пользователя (`components/AuthForm.tsx`)
- **Сейчас:** Данные авторизации обрабатываются локально. При клике на «Bypass» или успешной отправке формы вызывается проп `onSuccess`, передающий объект пользователя в стейт `App`.
- **Что сделать:**
  - Настроить отправку POST-запроса на `/api/auth/login` и `/api/auth/register`.
  - При регистрации отправлять выбранные роли волонтера (`selectedRoles`), телефон и выбранную гео-зону.
  - Сохранять полученный сессионный токен / Mesh-ID в LocalStorage / Cookies.
  - Реализовать авто-логин при загрузке страницы, если токен присутствует.

### 2. Отправка сигналов SOS (`main.tsx` -> `modalType === 'SOS'`)
- **Сейчас:** Вызывается стандартный `alert()`.
- **Что сделать:**
  - При подтверждении отправки SOS делать POST-запрос на `/api/emergency/sos` или слать сообщение в WebSocket / локальный UDP-сокет mesh-сети.
  - Передавать координаты пользователя (в симуляторе захардкожены `50.0755° N, 14.4378° E`, нужно получать через `navigator.geolocation` или запрашивать у локального GPS-модуля ноды).
  - Передавать телефон и имя пострадавшего.

### 3. Регистрация волонтерской помощи (`main.tsx` -> `modalType === 'VOLUNTEER'`)
- **Сейчас:** Локальные заглушки вызывают `alert()`.
- **Что сделать:**
  - Отправлять выбранный тип помощи (транспорт, физ. сила) на эндпоинт `/api/volunteer/register`.
  - Обновлять статус волонтера в профиле на сервере, чтобы другие участники сети видели активных помощников на карте или в списках координации.

### 4. Местные каналы новостей и сообщений (`main.tsx` -> `LATEST_MESSAGES`)
- **Сейчас:** Захардкоженный список сообщений в памяти.
- **Что сделать:**
  - Загружать список сообщений с эндпоинта `/api/messages` при старте и по таймеру (или по событию синхронизации).
  - В случае полной автономности — читать сообщения из локальной БД (IndexedDB / SQLite), которую наполняет фоновый mesh-демон.
  - Добавить отправку сообщений пользователем в разделы mesh-каналов (POST-запрос `/api/messages/create`).

### 5. Интерактивная карта и маркеры (`components/MapCard.tsx` и `MAP_FULL`)
- **Сейчас:** Точки отрисованы статично через SVG с фиксированными координатами.
- **Что сделать:**
  - Настроить получение массива точек интереса (POI) с бэкенда через `/api/map/markers` (координаты, тип маркера — вода/аптека/база, статус, описание).
  - Динамически рендерить маркеры на карте на основе полученного массива.
  - Интегрировать реальный картографический движок (например, Leaflet / Mapbox / Openlayers с оффлайн-тайлами), если потребуется полноценное гео-позиционирование вместо симулятора.

### 6. Статистика Mesh-соединения (`main.tsx` -> `modalType === 'SYNC'`)
- **Сейчас:** Статические цифры (5 активных нод, 14.2 МБ размер карты, время 08:12).
- **Что сделать:**
  - Запрашивать статус соединения у локального mesh-агента (например, `/api/mesh/status`).
  - Выводить количество реально обнаруженных пиров (active nodes) и время последней успешной репликации пакетов данных.

### 7. Чат с оффлайн AI-ассистентом (`main.tsx` -> `modalType === 'AI'`)
- **Сейчас:** Статический диалог-симуляция в коде.
- **Что сделать:**
  - Связать поле ввода с отправкой POST-запроса на `/api/ai/chat` (где запрос проксируется на локальный запущенный экземпляр Ollama / Llama.cpp на этом же устройстве).
  - Реализовать стриминг ответа (Server-Sent Events или WebSocket) для плавного вывода текста.
  - Хранить историю диалога в локальном стейте модального окна.
